# Управление памятью и ресурсами в Juno

В Juno реализована модель управления памятью, сочетающая низкоуровневый доступ к ресурсам с механизмами статической верификации для предотвращения распространенных ошибок системного программирования.

## Динамическая память (Heap)

Работа с кучей осуществляется через стандартные примитивы аллокации. Компилятор не использует неявную сборку мусора (Garbage Collection), что гарантирует детерминированное время выполнения.

*   `malloc(size: int): ptr` — выделяет блок памяти размером `size`. Возвращает указатель или `0` при неудаче.
*   `free(ptr: ptr)` — освобождает блок памяти, на который указывает `ptr`.

## Статический аудит ресурсов (Resource Auditor)

Одной из ключевых особенностей Juno является встроенный анализатор жизненного цикла ресурсов. Он работает на этапе компиляции и анализирует пути выполнения функций.

### Механизм работы
1.  **Регистрация**: Когда переменной присваивается результат функции, создающей ресурс (`malloc`, `open`), анализатор помечает её как "активный ресурс".
2.  **Отслеживание**: Анализатор проверяет все ветки выполнения на наличие вызовов функций освобождения (`free`, `close`).
3.  **Верификация**: Если при выходе из функции ресурс остается активным (не был освобожден или передан далее), компилятор генерирует предупреждение или ошибку.

> **Примечание**: Resource Auditor учитывает алиасинг (копирование указателей), позволяя отслеживать ресурс через разные переменные внутри одной функции.

## Работа со стеком (Stack)

Стек используется для хранения локальных переменных, аргументов функций и контекста выполнения.

*   **Выравнивание**: Согласно System V ABI, указатель стека (`rsp`) всегда выравнивается по границе 16 байт перед вызовом других функций.
*   **Размещение**: Массивы фиксированного размера, объявленные локально, размещаются в стеке. Если адрес локальной переменной передается вовне (через `&`), компилятор может выдать предупреждение об опасности использования указателя после выхода из области видимости.

## Арифметика указателей

Указатели в Juno являются типизированными. При использовании арифметических операций размер шага автоматически масштабируется в зависимости от типа данных, на который указывает указатель.

```juno
let p: ptr = malloc(16)
let next = p + 1            // Смещение на 8 байт (размер слова)
```

> **Важно**: Для работы с байтовыми смещениями используйте приведение типа к `u8*` или встроенные функции `ptr_add_bytes`.

## Компактное представление данных (Alignment)

Поля в структурах выравниваются по их естественной границе (например, 8-байтовые `int` выравниваются по 8 байтам). Для управления упаковкой данных используйте модификатор `packed` при определении структуры.

```juno
packed struct Header {
    let type: u8
    let length: u32
}
```
> **Внимание**: Использование упакованных структур может привести к снижению производительности на некоторых архитектурах из-за невыровненного доступа к памяти.
