# Управление памятью

Juno предоставляет низкоуровневый контроль над памятью, характерный для системных языков, но дополняет его статическими проверками безопасности.

## 1. Автоматический аудит ресурсов (Ownership)

Компилятор Juno отслеживает жизненный цикл ресурсов (указателей), созданных через функции аллокации.

**Основные правила:**
- **Создание**: Вызов `malloc` или функций, создающих объекты (например, `json_parse`), помечает переменную как "владельца" ресурса.
- **Передача**: Передача указателя в другую функцию может передать владение. После этого использовать переменную в исходной функции нельзя.
- **Освобождение**: Каждый созданный ресурс должен быть освобожден (`free`) или передан другой функции. Если ресурс "теряется" при выходе из области видимости, компилятор выдаст ошибку.

## 2. Аллокация в куче (Heap)

Juno использует системный аллокатор или встроенную реализацию поверх `mmap`.

- `malloc(size)`: Выделяет блок памяти указанного размера.
- `free(ptr)`: Освобождает выделенную память.

## 3. Стек и регистры (Stack & Registers)

- **Регистры**: Компилятор старается держать локальные переменные в кали-сейв регистрах процессора для максимальной скорости.
- **Стек**: Локальные массивы (`let arr[5]`) и переменные, адрес которых берется через `&`, размещаются в стеке.
- **Alignment**: Стек всегда выравнивается по границе 16 байт перед вызовом других функций (соответствие System V ABI).

## 4. Массивы и Указатели

Массивы в Juno — это непрерывные блоки памяти.
```swift
let arr[10]      // Массив на стеке
let p = &arr[0]  // Указатель на начало
let val = *p     // Разыменование
```
Использование `ptr_add(p, offset)` позволяет выполнять арифметику указателей с учетом размера типа.
