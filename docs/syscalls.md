# Системные вызовы (Syscalls)

Juno обеспечивает прямой доступ к ядру Linux без промежуточных слоёв.

## Использование `syscall`

Вы можете вызывать любые системные функции напрямую через встроенную функцию `syscall`. Она автоматически распределяет аргументы по нужным регистрам процессора.

```swift
// write(1, "Hi", 2)
syscall(1, 1, "Hi", 2)
```

## Соответствие регистрам (Linux x86-64)

При выполнении `syscall` аргументы распределяются следующим образом:
1.  Номер вызова: `RAX`
2.  Arg 1: `RDI`
3.  Arg 2: `RSI`
4.  Arg 3: `RDX`
5.  Arg 4: `R10`
6.  Arg 5: `R8`
7.  Arg 6: `R9`

Результат вызова всегда возвращается в `RAX`.

## Встроенные обертки

Для удобства в компиляторе реализованы именованные обертки для часто используемых вызовов:
- `getpid()`, `getuid()`
- `mmap(addr, len, prot, flags, fd, off)`
- `munmap(addr, len)`
- `fork()`
- `execve(path, argv, envp)`
- `mkdir(path, mode)`, `rmdir(path)`
- `chmod(path, mode)`

Эти функции вызывают `syscall` внутри, обеспечивая более читаемый код.
