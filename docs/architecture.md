# Архитектура Juno

Компилятор Juno спроектирован как модульный конвейер, работающий напрямую с абстрактным синтаксическим деревом (AST) без промежуточного представления (IR), что обеспечивает высокую скорость компиляции и предсказуемость результата.

## Компоненты конвейера

### 1. Фронтенд (Frontend)
- **Lexer**: Токенизатор, превращающий текст в поток типизированных токенов.
- **Parser**: Рекурсивный нисходящий парсер. Генерирует AST. Поддерживает методы структур, дженерики и современный синтаксис.
- **Preprocessor**: Обрабатывает директивы условной компиляции (например, для разных ОС или архитектур).
- **Importer**: Разрешает зависимости между файлами и подключает модули стандартной библиотеки.

### 2. Мидл-энд (Middle-end)
- **Monomorphizer**: Выполняет специализацию дженериков. Заменяет полиморфные вызовы и структуры конкретными реализациями для каждого типа данных.
- **Semantic Analyzer**: Проверяет область видимости переменных, типы данных и сигнатуры функций.
- **Resource Auditor**: Проводит статический анализ графа владения ресурсами, выявляя утечки памяти (Memory Leaks) и ошибки повторного освобождения.
- **Turbo Optimizer**: Выполняет базовые оптимизации, такие как свертка констант (Constant Folding) и оптимизация хвостовой рекурсии.

### 3. Бэкенд (Backend)
- **Register Allocator**: Использует алгоритм Greedy Linear Scan для распределения переменных по аппаратным регистрам (RBX, R12-R15 на x86; X19-X28 на ARM).
- **Native Generator**:
    - **Control Flow**: Генерирует переходы, циклы и условия.
    - **ABI Compliance**: Управляет выравниванием стека (16 байт) и сохранением регистров согласно System V ABI.
    - **Emitter**: Непосредственно кодирует инструкции x86-64 и AArch64 в байты.

### 4. Линковка (Linker)
- Собирает секции кода (`.text`), данных (`.data`) и неинициализированных данных (`.bss`).
- Разрешает адреса (Relocation) для глобальных переменных, строк и вызовов функций.
- Формирует финальный исполняемый файл (ELF для Linux).
