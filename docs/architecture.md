# Архитектура компилятора Juno

Компилятор Juno реализован как многопроходной конвейер трансляции исходного кода в исполняемые бинарные файлы. Архитектура исключает использование тяжелых внешних зависимостей (LLVM/GCC), обеспечивая прямой контроль над каждым этапом кодогенерации.

## Конвейер компиляции

### 1. Лексический и синтаксический анализ (Frontend)
*   **Lexer**: Выполняет токенизацию исходного текста. Поддерживает многобайтовые кодировки в строках и сложную обработку числовых литералов (hex, binary, float).
*   **Parser**: Реализует алгоритм рекурсивного нисходящего парсинга. Результатом является абстрактное синтаксическое дерево (AST), представляющее структуру программы.
*   **Importer**: Управляет графом зависимостей. Реализует поиск модулей в путях стандартной библиотеки и пользовательских директориях, исключая циклические импорты.

### 2. Семантическая обработка (Middle-end)
*   **Monomorphizer**: Подсистема обработки обобщенного программирования (Generics). На основе использования в коде генерирует уникальные реализации функций и структур для конкретных типов данных.
*   **Semantic Analyzer**: Выполняет проверку типов, разрешение областей видимости (Scoping) и валидацию сигнатур функций. Гарантирует корректность типов перед переходом к генерации кода.
*   **Resource Auditor**: Модуль статической верификации. Анализирует граф вызовов на предмет корректности работы с динамической памятью и дескрипторами файлов.

### 3. Оптимизация и распределение ресурсов
*   **Turbo Optimizer**: Выполняет трансформации на уровне AST:
    *   **Constant Folding**: Вычисление константных выражений во время компиляции.
    *   **Inlining**: Подстановка тел функций для устранения накладных расходов на вызов.
    *   **Dead Code Elimination**: Удаление недостижимых участков кода.
*   **Register Allocator**: Основан на алгоритме **Linear Scan**. Распределяет переменные по регистрам процессора, минимизируя операции `push/pop` и обращения к стеку.

### 4. Генерация кода и линковка (Backend)
*   **Native Generator**: Транслирует оптимизированное AST в машинные коды. Учитывает специфику целевой архитектуры (наборы инструкций, способы адресации).
*   **Emitter**: Низкоуровневый компонент, отвечающий за кодирование конкретных инструкций (x86-64 REX-префиксы, ModRM-байты; AArch64 фиксированные 32-битные опкоды).
*   **ELF Builder**: Формирует структуру исполняемого файла. Создает заголовки (EHDR, PHDR), секции кода и данных, а также таблицы символов.

## Специфика реализации бэкендов

| Архитектура | Модель вызовов | Особенности |
| :--- | :--- | :--- |
| **x86-64** | System V AMD64 ABI | Аргументы в RDI, RSI, RDX, RCX, R8, R9. Выравнивание стека 16 байт. |
| **AArch64** | ARM 64-bit ABI | Аргументы в X0-X7. Использование регистра связи (LR) для возврата из функций. |

> **Заметка для разработчика**: Отсутствие промежуточного представления (IR) в текущей версии Juno упрощает отладку компилятора, но накладывает ограничения на кросс-блочные оптимизации, которые планируются в будущих версиях.
