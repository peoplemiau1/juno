# CLI и Сборка

## Основные команды
Используйте унифицированный интерфейс `juno` для управления проектом.

*   `juno <file.juno>`: По умолчанию собирает файл для текущей архитектуры (x86_64).
*   `juno build <file.juno>`: Аналог выше.
*   `juno run <file.juno>` или `juno -r <file.juno>`: Скомпилировать и сразу запустить.
*   `juno -a aarch64 <file.juno>`: Скомпилировать для ARM64.
*   `juno -o <path> <file.juno>`: Указать путь выходного файла.
*   `juno --hell [level] <file.juno>`: Включить режим обфускации (easy, medium, hard, nightmare, hell).
*   `juno version`: Показать версию и логотип.
*   `juno help`: Справка по всем флагам.

## Опции
| Флаг | Короткий | Описание |
|------|----------|----------|
| `--arch` | `-a` | Архитектура: `x86_64`, `aarch64` |
| `--target` | `-t` | ОС: `linux`, `windows`, `flat` |
| `--output` | `-o` | Путь к бинарнику |
| `--run` | `-r` | Запуск после сборки |
| `--hell` | | Уровень обфускации |

## Выходные форматы
*   **Linux (ELF64)**: Статическая сборка, не требует libc.
*   **Windows (PE64)**: Поддержка `.exe`.
*   **Flat (Binary)**: Чистый машинный код для загрузчиков или микроконтроллеров.

## Подсказки (Did you mean?)
Если вы опечатались в команде или флаге, CLI Juno предложит наиболее вероятный вариант.
Пример: `juno --arch arm64` -> `Did you mean? aarch64`.
