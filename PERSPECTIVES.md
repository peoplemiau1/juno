# Перспективы развития языка Juno (+)

## 1. Достижения
- **Регистровая аллокация**: Переход от чисто стековой машины к современной реестровой архитектуре.
- **Поддержка AArch64**: Полная поддержка ARM64 Linux, включая Termux и мобильные устройства.
- **Ownership Audit**: Статическая проверка безопасности памяти.

## 2. Краткосрочные цели (v2.2 - v2.5)
- **Graph Coloring Allocator**: Дальнейшее улучшение использования регистров.
- **SIMD Intrinsics**: Прямой доступ к AVX2/SSE4 (x86) и NEON (ARM) на уровне языка.
- **Нативный TLS**: Избавление от зависимости от `curl`.

## 3. Долгосрочные цели ()
- **Self-Hosting**: Переписывание компилятора Juno на самом Juno. Это станет финальным тестом зрелости языка.
- **LLVM Backend**: Опциональный бэкенд для доступа к оптимизациям мирового уровня.
- **Standard Library Expansion**: Полноценные HTTP-фреймворки, работа с криптографией и базами данных.

## 4. Экосистема
- **JunoPM**: Пакетный менеджер для Juno.
- **LSP**: Language Server Protocol для поддержки в VS Code и Neovim.
- **Juno Playground**: Онлайн-компилятор для тестирования языка в браузере.

## 5. Философия
Juno остается быстрым, маленьким и прозрачным. Мы не планируем раздувать язык лишним синтаксическим сахаром, фокусируясь на нативной производительности и безопасности ресурсов.
