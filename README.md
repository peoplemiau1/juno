## Техническая спецификация v2.1

Juno — это язык программирования, спроектированный для написания критически важного системного ПО, сетевых сервисов и средств обеспечения безопасности. Версия 2.1 вводит строгие архитектурные гарантии и оптимизации промышленного уровня.

### Ключевые компоненты архитектуры

*   **Реестровая аллокация (Linear Scan)**: Отказ от стековой модели в пользу прямого маппинга переменных на физические регистры CPU (`RBX, R12-R15` на x86; `X19-X28` на ARM). Минимизация `memory-to-register` трафика.
*   **Ownership & Resource Audit (E0007)**: Статический анализатор жизненного цикла ресурсов. Enforcing семантики владения: автоматическая детекция утечек (Leaks) и висячих указателей (Use-After-Free) на этапе компиляции.
*   **Нативный кодогенератор**: Прямая генерация машинных инструкций без промежуточного байт-кода или использования LLVM. Полная автономность и предсказуемость бинарного выхлопа.
*   **Metamorphic Engine (Hell Mode)**: Продвинутая система обфускации с прецизионным декодером длин инструкций. Динамическая мутация кода без нарушения логической целостности.

## Развертывание

```bash
git clone https://github.com/peoplemiau1/juno.git
cd juno && ./install.sh && source ~/.bashrc
```

## Использование (CLI Interface)

```bash
# Компиляция с агрессивной оптимизацией
juno build main.juno -o bin_x64

# Кросс-компиляция для архитектуры ARM64 (AArch64)
juno build main.juno -a aarch64 -o bin_arm

# Сборка защищенного бинарника (Metamorphic Mode)
juno build main.juno --hell nightmare
```

## Стандартная библиотека v3 (Core Modules)

| Модуль | Описание |
|--------|----------|
| `std/json` | Высокопроизводительный парсер JSON с поддержкой объектов и массивов. |
| `std/net` | Низкоуровневые обертки над сокетами, поддержка Epoll/IO_Uring паттернов. |
| `std/str` | Управление динамическими строками с автоматическим контролем границ. |
| `std/arena` | Детерминированный регионный аллокатор для высоконагруженных систем. |
| `std/fs` | Интерфейс взаимодействия с VFS Linux. |

## Безопасность и Стабильность

*   **ABI Compliance**: Строгое соблюдение 16-байтового выравнивания стека согласно System V ABI.
*   **Callee-Saved Preservation**: Автоматическая генерация прологов и эпилогов для сохранения состояния вызывающей функции.
*   **Unified Error Handling**: Структурированные сообщения об ошибках с указанием точного смещения в исходном коде.
